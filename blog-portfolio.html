How I Built My Portfolio.

Building a personal website is a rite of passage for every developer. Over the years, I've gone through multiple iterations, jumping from React to Next.js, and even trying out massive, complex templates that ultimately felt like they didn't belong to me.

For this version, I decided to step back. The goal was simple: minimal markup, zero heavy frameworks, and a deep focus on typography and aesthetic details. No complicated build steps. Just raw HTML, CSS, and a sprinkle of JavaScript.

The CSS Architecture

Instead of reaching for Tailwind or CSS-in-JS, I wrote a custom CSS file leveraging modern CSS variables. This allowed me to create a highly flexible dark mode toggle without requiring any runtime JavaScript calculation beyond flipping a single attribute on the `<html>` tag.

I also heavily relied on `clamp()` for responsive typography. This little CSS function makes the fonts fluidly scale between mobile and desktop without needing a dozen media queries.

Why No Framework?

In 2026, it's almost controversial to say you didn't use a framework. But for a portfolio—a static document with minimal state—a framework is often overkill. By removing it, my site loads instantaneously, the bundle size is virtually non-existent, and the code is incredibly easy to read and maintain.

Sometimes, the best engineering decision you can make is to know when to keep things simple.
